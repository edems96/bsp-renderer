Introduction

bsp-renderer is an open source application for displaying levels stored in the quake3 bsp map format. It allows the user to move around in the levels using and first person camera and collision detection. The purpose of this document is to give an overview of the bsp-renderer application, including how to use it, which third party libraries and third party code it uses and other technical information. The downloadable distribution of the program contains the source code and prebuilt executables. It also contains the libraries necessary to build the source, and a Quake III level called "The Forlorn Hope", which was found at http://www.users.on.net/justonefix/. bsp-renderer is currently in version 0.7.

How to run the application

The project contains two configurations, debug and release. The prebuilt executable is a release build, which has the flag RELEASE_BUILD set. This flag affects were the application looks for the default map. If the application finds a bsp.cfg file in its running directory, it will parse this file to set which map to load. See the example file for an explanation of the syntax. If no file is present, or no file or map is set in the file, the default map will be loaded ("The Forlorn Hope" found in the 'media' folder). The application will exit if it cannot find the map or data file bsp.cfg specifies. The runtime library zlib1.dll needs to be in the run directory, it it does not exist in the default Windows dll directories.

Keys

Use WASD and mouse to move around.
F1: disable collision detection.
F2: enable collision detection without gravity
F3: enable collision detection with gravity
F8: toggle full screen mode

Licensing

bsp-renderer is released under GPL version 3. However, it relies on third party libraries, as well as pieces of source code from other open source programs. An overview of these libraries and source code is given here. The licenses mentioned can be found in the 'licenses' folder.

Source code licenses

Two parts of the source code are based on algorithms originally from other open source applications.

- The source for the collision detection (but not the collision response) is based on code from reference [3] above, which in turn comes from the Quake III source.

- The tessellation of the Bezier patches is based on code from reference [1]. The code originally comes from the Octagon Engine.
